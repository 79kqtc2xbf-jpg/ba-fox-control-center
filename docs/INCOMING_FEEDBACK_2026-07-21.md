# Входящая обратная связь по трекеру — 21.07.2026

Источник: комментарии коллег к первой версии трекера и файл «ОС по трекеру». Это список для приоритизации; изменения кода по нему ещё не выполнены.

## Продуктовый backlog

| Тема | Наблюдение | Предлагаемое направление | Статус |
| --- | --- | --- | --- |
| Стартовая страница | На desktop нарушена разметка | Воспроизвести на целевых разрешениях, исправить адаптивную сетку, добавить регрессионную UI-проверку | Требует triage |
| Контакты | Ручной ввод контакта в задаче лишний либо поле не обязательно | Поле уже не обязательное; в UI это явно обозначено, а пустое значение не показывается в карточке. Подсказки из `Contacts` и правила выбора — отдельная задача | Частично сделано |
| Плотность интерфейса | Слишком много блоков, цветов, размеров и быстрых действий | В карточке задачи второстепенные действия и служебные данные перенесены в «Ещё». Дальше — проверить и упростить остальные экраны, убрать дубли | Частично сделано |

| Метки | Нужны ярлыки и поиск по ним | Спроектировать task tags, фильтры и миграцию | Требует схемы |
| Декомпозиция | Нужны задачи верхнего уровня и подзадачи с наглядной структурой | Спроектировать `parentTaskId`, дерево, статусы и правила агрегации прогресса | Требует схемы и UX |
| Учёт времени | Нужна статистика фактического времени на задачу | Сначала выбрать простой MVP: ручной таймер/старт-стоп или внесение длительности; определить хранение и отчёт | Требует продуктового решения |

## Рекомендуемый порядок

1. Исправить desktop-адаптацию и уменьшить перегруженность основного экрана.
2. Согласовать модель списка: активные, выполненные, архив и детали задачи.
3. Спроектировать расширение схемы единым пакетом: контакты, теги, родительская задача, приоритет и время.
4. Только после согласования выполнить миграцию Google Sheet и UI-реализацию по небольшим PR.

## Ограничения для будущей реализации

- Не добавлять колонки в рабочую Google Sheet автоматически.
- Перед миграцией сделать и проверить резервную копию, описать откат и получить явное согласование владельца продукта.
- Сохранить доступность существующих задач и legacy-полей.
- Не смешивать этот backlog с изменениями авторизации, ролей или видимости.
