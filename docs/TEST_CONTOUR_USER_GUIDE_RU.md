# MF Group Tracker — инструкция для тестового контура

## 1. Что это

MF Group Tracker — тестовый операционный дашборд для просмотра и ведения командных задач.

Ссылка на тестовый контур:

https://79kqtc2xbf-jpg.github.io/ba-fox-control-center/

Для работы не нужны GitHub, GitHub-токены, установка программ или доступ к исходному коду.

## 2. Кто может войти

Войти можно только рабочим Google-аккаунтом `@mfstream.io`, который заранее добавлен в реестр пользователей.

Если страница открылась, но запись недоступна:

1. Откройте раздел **«Настройки»**.
2. Нажмите **«Войти через Google»**.
3. Выберите рабочий аккаунт `@mfstream.io`.
4. Дождитесь сообщения **«Профиль подтверждён»**.
5. Проверьте, что сверху появилась надпись **«Запись включена»**.

Google-вход иногда подтверждается до 30 секунд. Не нажимайте кнопку несколько раз подряд.

## 3. Быстрый тест за 5 минут

Не изменяйте реальные задачи коллег. Для проверки создайте отдельную тестовую задачу.

1. Нажмите **«Новая задача»**.
2. Назовите её по шаблону:

   `TEST_ИМЯ_ДАТА_DO_NOT_USE`

3. Укажите себя ответственным.
4. Добавьте короткий следующий шаг и контрольную дату.
5. Сохраните задачу и дождитесь подтверждения.
6. Найдите её в разделе **«Все задачи»**.
7. Последовательно проверьте:

   - перевод в статус **«В работе»**;
   - напоминание **«+3 days»**;
   - кнопку **«Обновить этап»**;
   - завершение через **«✓ Выполнено»**.

8. После теста обязательно отметьте QA-задачу выполненной.

Запись и обновление данных могут занимать 10–30 секунд. Не повторяйте действие, пока кнопки заблокированы или отображается индикатор сохранения.

## 4. Основные разделы

### Обзор

Показывает общую картину: активные задачи, просрочку, ожидания, блокеры и задачи без ответственного.

### Мои задачи

Показывает задачи, связанные с текущим пользователем. В тестовом режиме персональная фильтрация ещё не является окончательной.

### Мой фокус

Собирает задачи, которые вручную добавлены в фокус или требуют повышенного внимания.

### Все задачи

Общий список задач. Здесь удобнее всего искать созданную тестовую задачу.

### Кто чем занят

Показывает распределение активных задач и просрочки по ответственным.

### Отделы / направления

Группирует задачи по текущей категории или направлению.

### Риски и зависшие

Показывает просроченные задачи, ожидания и другие элементы, требующие внимания.

### Отчёты

Формирует предварительный текст отчёта. Автоматическая отправка отчётов коллегам пока не включена.

### Настройки

Содержит Google-вход, информацию о профиле, роли, тестовую проверку видимости и локальные настройки оформления.

## 5. Что уже работает

- просмотр live-данных из Google Sheets;
- безопасный вход через рабочий Google-аккаунт;
- создание тестовой задачи;
- изменение статуса;
- перенос напоминания;
- обновление этапа, комментария, следующего шага и контрольной даты;
- добавление задачи в фокус;
- завершение задачи;
- запись автора и ответственного в служебные identity-поля;
- автоматический аудит действий с ID задачи и значениями до/после;
- роли `admin`, `executive` и `member`;
- защита записи: без подтверждённого Google-профиля кнопки изменения недоступны.

## 6. Что пока не работает или ограничено

- Персональная фильтрация задач по сотрудникам не включена. Во время теста участники видят общий дашборд.
- Тестировщик не должен изменять реальные задачи, если это не согласовано с Лизой.
- Удаление задач и полноценное архивирование через веб-интерфейс не предусмотрены.
- Кнопка **«Блокер»** пока недоступна.
- Автоматическая отправка отчётов, Telegram-уведомлений, Gmail-операции и WhatsApp-интеграция в этот тест не входят.
- Изменение темы и цвета интерфейса хранится только в текущем браузере.
- Обновление live-данных может отображаться с задержкой до 60 секунд.
- Разделы видимости и ролей работают в режиме проверки; принудительная фильтрация по пользователю выключена.

## 7. Правила безопасного тестирования

- Создавайте только задачи с префиксом `TEST_`.
- Не используйте реальные персональные, финансовые или конфиденциальные данные.
- Не редактируйте чужие рабочие задачи без согласования.
- Не передавайте ссылку людям, которых нет в согласованной тестовой группе.
- Не отправляйте никому токены, пароли или доступ к GitHub — они для теста не требуются.
- После проверки завершайте тестовую задачу.

## 8. Если что-то не работает

Сначала:

1. Подождите 30 секунд.
2. Нажмите **«Обновить»** один раз.
3. Проверьте, что вверху указано **«Запись включена»**.
4. Откройте **«Настройки»** и убедитесь, что профиль подтверждён через Google.

Если проблема осталась, отправьте Лизе:

- ваше имя и рабочий email;
- время возникновения ошибки;
- название или ID тестовой задачи;
- какое действие выполнялось;
- текст ошибки;
- скриншот экрана;
- браузер и устройство.

Не отправляйте пароли, Google-токены или содержимое адресной строки Apps Script.

## 9. Критерий успешного теста

Тест считается успешным, если пользователь:

1. вошёл через рабочий Google-аккаунт;
2. увидел live-дашборд;
3. создал отдельную тестовую задачу;
4. изменил статус;
5. перенёс напоминание;
6. обновил этап;
7. завершил задачу;
8. не столкнулся с потерей или дублированием данных.
