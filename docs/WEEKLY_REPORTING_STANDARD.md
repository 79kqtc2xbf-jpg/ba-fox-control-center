# BA Fox — Weekly Reporting Standard

## Purpose

This document fixes the standard for Lisa's weekly EA / BA Fox reports.

Weekly reports must be broader than daily reports and should show ownership, progress, closure, and controlled risk management.

The goal is to avoid the impression that tasks are endlessly dragging. The report should show what moved, what was closed, what was converted into controlled follow-up, and what is blocked with a clear next action.

## Core weekly framing

A weekly report should not look like:

```text
I pinged / I was waiting / still in progress / no response
```

It should look like:

```text
Result of the week → current state → next action → owner/control date
```

## Recommended weekly structure

Use this structure by default:

```text
#итоги_недели #ea
[week dates]

1. Executive summary
2. Closed / finalized this week
3. Moved forward / under control
4. Blockers and managed risks
5. Wait list / external dependencies
6. Key outcomes by workstream
7. Focus for next week
8. What I need / decisions needed
```

## 1. Executive summary

Start with a short owner-style summary of the week.

Purpose:

- show that Lisa controlled multiple workstreams;
- highlight actual movement;
- set context before details;
- reduce the feeling of scattered or unfinished tasks.

Example:

```text
На этой неделе я закрыла и продвинула ключевые блоки по документам, банковскому онбордингу, партнёрским коммуникациям и презентациям. Основной фокус был на том, чтобы не просто вести переписки, а переводить задачи в финальные статусы: где возможно — закрывать, где есть внешний блокер — фиксировать следующий шаг и срок контроля.
```

## 2. Closed / finalized this week

Use for tasks with clear visible result or near-final result.

Format:

```text
• [Task / organization] — [result] → [what remains if any]
```

Examples:

```text
• Bitazza — пакет KYB собран примерно на 70%; остались финальные коррекции и отправка.
• E-pay — документы собраны и переданы.
• Презентации — доработаны и готовы к отправке / отправлены партнёрам.
• Документы для Александры — подготовлены.
```

## 3. Moved forward / under control

Use for tasks that are not closed but visibly progressed.

Format:

```text
• [Task / organization] — [what moved] → [next action] → [control date]
```

Examples:

```text
• BCEL — коммуникация по PromptPay переведена в follow-up; контроль ответа до понедельника.
• P3 Estates — коммуникация переведена в дожим конкретного слота встречи.
• GLN — основной канал не дал результата, поэтому перехожу через альтернативные контакты.
```

## 4. Blockers and managed risks

Use for blockers, but never as passive excuses.

Format:

```text
• [Task] — [blocker] → [what I am doing] → [next check / decision date]
```

Examples:

```text
• Лаос / офис — оплата за февраль–март упирается в отсутствие денег на корпоративной карте; контроль и повторный запуск в понедельник.
• Пошлина США — оплата не проходит; повторно написала в поддержку и параллельно ищу альтернативный способ оплаты.
• BOL — ожидаем решение по переводу; следующий этап подготовлен и стартует сразу после получения.
```

## 5. Wait list / external dependencies

Use only for fixed external dependencies where Lisa does not own the next direct action.

Examples:

```text
• Flow — подписание в Минюсте.
• JDB — проверка документов + открытие QR.
• JDB — виртуальная карта: зафиксировано, что у них её нет.
```

## 6. Key outcomes by workstream

For weekly reports, group details by workstream if the report becomes long.

Recommended workstreams:

```text
📄 Документы / онбординг
💌 Почта / партнёрские коммуникации
🏦 Банки / платежи
🏡 Недвижимость / партнёры
⚙️ Автоматизация / BA Fox
🤝 HR / организационные задачи
⏳ Wait list / внешние зависимости
```

Each workstream should contain 2–5 bullets maximum.

## 7. Focus for next week

End with 3–5 priorities for next week.

Format:

```text
На следующей неделе фокус:
• [priority 1] — [desired outcome]
• [priority 2] — [desired outcome]
• [priority 3] — [desired outcome]
```

Example:

```text
На следующей неделе фокус:
• Закрыть Bitazza отправкой полного пакета.
• Перевести BCEL / Super Rich / GLN в конкретные ответы или встречи.
• Разблокировать платежи по Лаосу и пошлине США.
• Продвинуть JDB и Kasikorn до следующего подтверждённого шага.
```

## 8. What I need / decisions needed

Use this block only when needed.

Purpose:

- make dependencies visible;
- avoid taking blame for decisions or resources outside Lisa's control;
- give Theodor/manager clear points where input is required.

Examples:

```text
Нужно решение / подтверждение:
• Пополнение корпоративной карты для оплаты Лаоса.
• Подтверждение приоритета: Bitazza vs Kasikorn vs PayConnect.
• Решение по дальнейшей коммуникации с Shobana / India.
```

## Weekly tone rules

The weekly report should sound:

- calm;
- operational;
- confident;
- not defensive;
- not overloaded with excuses;
- focused on outcomes and next actions.

## Forbidden weekly framing

Avoid standalone phrases:

```text
много пинговала
всё тянется
жду ответы
не успела
нет ответа
в процессе
```

Use instead:

```text
перевела в контроль до [date]
зафиксировала внешний блокер
перешла на альтернативный канал
закрыла текущий этап
следующий контроль назначен на [date]
ожидаю внешнее решение, следующий шаг подготовлен
```

## Chat organization decision

A separate project chat should be used for weekly EA reports.

Recommended chat title:

```text
BA Fox — Weekly EA Reports / Итоги недели
```

Alternative shorter title:

```text
BA Fox — Итоги недели EA
```

This chat is for:

- raw weekly notes;
- daily fragments for the weekly report;
- end-of-week synthesis;
- owner-style rewriting;
- next-week focus planning.

## Approved behavior

When Lisa sends a usual weekly report or raw notes, BA Fox should:

1. Preserve the facts.
2. Reframe the report into the weekly owner-style structure.
3. Remove the feeling that tasks are dragging.
4. Show which tasks were closed, progressed, controlled, blocked, or parked.
5. Add next actions and control dates where possible.
6. Keep the report broader and more strategic than the daily report.
